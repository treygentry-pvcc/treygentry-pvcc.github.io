Operating System Terminology
Term Definition
Operating System The software that controls the overall operation of a computer.
Job Queue A job que is a mass storage organizatino that jobs reside in while awaiting execution. Jobs are ordered in a first in first out fashion.
Interactive Processing A feature of an operating system that allows for a program to communicate with the user through remote terminals.
Time-sharing A feature of an operating system that provides service to multiple users simultaneously.
Multiprogramming A technique of implementing time sharing by diving time into tintervals and then each jobs execution is restricted to one interval at a time.
Application software Consists of the programs for performing tasks particular to the machines utilization.
System software Performs the tasks that are common to computer systems in general; it provides the infrastructure that the application software requires.
Utility software Collection of software untits that perform activities critical to computer installations but are not included with the operating system.
Graphical User Interface (GUI) A type of user interface where opbjects that are being maniputlated are reperesented pictorially on the display as icons.
Shell Older type of user interface that communicates with the user through textual messages using a keyboard and monitor screen.
Kernel The internal part of the operating system containing the software components that perform the very basic functions required by the computer installation.
Device Driver Software units that communicate witht the controllers to carry out operations on the peripheral devices attached to the machine.
File Manager Coordinates the use of hte mahines mass storage facilities. Keeps track of the all the file locations and who can access them as well as which portions of mass storage available for new files.
Memory Manager Coordinates the use of the machines main memory.
Virtual Memory "fictional" memory space created by the process of paging or rotating data and programs back and forth between main memory and mass storage.
Boot Strapping Process(sometimes shortened to just booting) of starting the machine up in which the operating system is transferred from mass storage into main memory.
ROM Read only memory; the contents of the memory can be read but not altered.
Deadlock Condition in which 2 or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
Spooling A technique for avoiding deadlock by holding data for output at a later time thus allowing a non shareable resource to become a shareable one by simply giving the held data to the resource once that resource actually becomes available.
Super User AKA the administrator; a person with highly priveleged acess to the OS that can alter the operating systems settings, software packages, and the privleges of other users.
Multi-factor Authentification Electronic Authentification method where users must provide 2 or more distinct pieces of "evidence" (data only an authorized user would know i.e. a password)
FIFO (first in first out) refers to the method of ordering inside a Queue in which jobs are removed from the Queue in the order in which they arrive
